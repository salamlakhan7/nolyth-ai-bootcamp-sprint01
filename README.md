## Repo Structure
Each Python fundamentals topic is kept in its own file, and each subsequent phase (FastAPI basics, Auth + structure) is kept in its own dedicated folder with its own testing documentation and screenshots - committed and pushed individually to maintain a clean, incremental commit history, as required by the sprint guidelines. Git/GitHub concepts are documented in detail in `GIT_WORKFLOW.md`.

## Progress Approach
Following an incremental build-and-commit workflow: one topic → one file/folder → one commit → push → next topic. No batch pushes.
